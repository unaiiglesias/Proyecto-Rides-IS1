UMLUseCase
Untitled :: Requirements :: Pay reservation

Description

Flujo de eventos: realizar pago

  1. El sistema muestra todas las solicitudes del viajero.
  2. El viajero escoge un viaje y realiza el pago.

Flujo alternativo

  • Si la solicitud de reserva no ha sido aceptada, no se puede realizar el pago.
  • Si el viajero no tiene suficiente dinero en su cuenta, el pago no se realiza y el sistema lo indica.
